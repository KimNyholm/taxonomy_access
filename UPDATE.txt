READ THIS FILE if you are updating from previous versions of taxonomy_access.module for Drupal v.4.5. If you are
installing taxonomy_access.module for the first time, you may ignore this file.
(Send comments to Keve. http://drupal.org/user/13163/contact )

You need to make sure that your database schema is current.

(1) I included an update script in taxonomy_access.module. 
    This update script will help you update the database from previous versions of Taxonomy Access (before 02/01/2006) to newer versions. (NO WARRANTIES! For security reasons, do not forget to backup your database, before applying this script) 

    To run the script, go to 'admin/settings/taxonomy_access/update' while logged in as an administrator.

(2) You also have to creat new table 'term_access_defaults'.

# ----------
# For MySQL:
# ----------

CREATE TABLE term_access_defaults (
  vid int(10) unsigned NOT NULL default '0',
  rid int(10) unsigned NOT NULL default '0',
  grant_view tinyint(1) unsigned NOT NULL default '0',
  grant_update tinyint(1) unsigned NOT NULL default '0',
  grant_delete tinyint(1) unsigned NOT NULL default '0',
  grant_create tinyint(1) unsigned NOT NULL default '0',
  grant_list tinyint(1) unsigned NOT NULL default '0',
  PRIMARY KEY  (vid,rid)
) TYPE=MyISAM;

-- -------------
-- For PostgreSQL:
-- -------------

CREATE TABLE term_access_defaults (
 vid integer  NOT NULL default '0',
 rid integer  NOT NULL default '0',
 grant_view smallint NOT NULL default '0',
 grant_update smallint NOT NULL default '0',
 grant_delete smallint NOT NULL default '0',
 grant_create smallint NOT NULL default '0',
 grant_list smallint NOT NULL default '0',
 PRIMARY KEY  (vid,rid)
);
